### 2.1 Detector surface — 60+ finding codes by cluster

**Schema cluster** — column types, range info, nominal vs ordinal:

| Code | Severity | Meaning |
|---|---|---|
| E9002 | Info | Type info; missingness not auto-inferable for non-Float columns without a null mask |
| E9020 | Info | Numeric range summary |
| E9021 | Info | Categorical cardinality summary |
| E9022 | Info | Bool prevalence summary |
| E9023 | Notice | Densely-packed Int column — looks like a label-encoded nominal, not an ordinal |

**Missingness cluster** — NaN counts, sentinels, conditional missingness:

| Code | Severity | Meaning |
|---|---|---|
| E9001 | Notice | n values missing (via null mask) |
| E9006 | Error | Null mask references row out of bounds |
| E9007 | Info | Sentinel numeric value (-9999 / 99999) at ≥1% |
| E9008 | Info | Sentinel string value (`?`/`NA`/`null`/`none`/…) at ≥1%; auto-folded into effective null mask |
| E9070 | Notice | Conditional missingness — `missing(A)` implies `missing(B)` in ≥95% of A's missing rows |

**Distribution cluster** — distinct counts, skew, outliers:

| Code | Severity | Meaning |
|---|---|---|
| E9010 | Notice | Column is constant (single value) |
| E9011 | Notice | Near-constant — dominant value ≥99% |

| E9080 | Warning | Case-fold collisions — `Premium`/`premium`/`PREMIUM` collapse under `str::to_lowercase` |

### 2.2 The public API surface

Every Locke entry point as a Python function or class:

#### Top-level functions

```python
import cjc_locke

# Single-DataFrame validation — runs every detector cluster above
report = cjc_locke.validate(data, label="dataset", options=None)
#   data: dict[str, ndarray|list] OR pandas/polars DataFrame
#   options: optional {"null_masks": {col: [row_indices]}, ...}
#   returns: LockeReport with severity_counts, finding_codes(),
#            to_json(), to_dict(), and run_id

# Train vs test drift — emits InductionRiskReport
drift = cjc_locke.compare_drift(train, test, drift_config=None)
#   drift_config: optional {"mean_shift_warn": 0.05, ...}

# Combined validate + drift + belief composition under meet-semilattice
val, drift, belief = cjc_locke.validate_and_compare(
    train, test, label="dataset", options=None, drift_config=None,
)

# Derive a belief report from a validation report
belief = cjc_locke.belief_report(report, model=None)
#   model: optional BeliefPenalty {"info": 0.0, "notice": 0.05, "warning": 0.15, "error": 0.30}

# Causal guardrail audit
cg = cjc_locke.causal_guardrail(
    data,
    target_column="income",
    causal_config={"mode": "observational_only"},  # or "default"
    label_text=None,
    interpret_model_explanation_as_causal=False,
)

# Temporal sanity (timestamps column expected as Int / DateTime)
codes = cjc_locke.detect_temporal_issues(data, time_col="ts", config=None)

# Policy gate — suppression/owner/requirement rules over a report
result = cjc_locke.apply_policy(report, policy={
    "suppressions": [{"code": "E9001", "column": "phone", "reason": "PII expected"}],
    "owners":       [{"team": "data-quality", "code": "E9072"}],
    "requirements": [{"code": "E9001", "operator": "eq_zero", "threshold": 0}],
})

# Canonical byte-identical JSON emit / parse
js = cjc_locke.emit_report_json(report)
restored = cjc_locke.parse_report_json(js)

# Audit event construction
ev = cjc_locke.make_audit_event("run-1", seq=5, kind="impression", subject_id=0xAAAA, note="...")
```

#### Streaming validator (out-of-core)

```python
sv = cjc_locke.StreamingValidator("stream", config={"sample_cap": 100_000})
for chunk in iter_chunks_somehow():
    sv.ingest_chunk(chunk)         # chunk is dict[str, ndarray|list]
final = sv.into_report()           # returns LockeReport
```

Caveat (B5.6 documented): `into_report()` is byte-identical to a
single-shot `validate(full_df)` only when no column exceeded
`sample_cap`. Past that point the report sees the truncated sample.
Running summaries (`streaming_summaries()`) are unaffected.

#### Lineage builder (audit chain)

```python
b = cjc_locke.LineageBuilder("daily-pipeline")
src = b.add_impression("train.csv", kind="dataset", n_rows=10_000, columns=["x", "y"])
filt = b.add_idea(name="filter_active", op_id="filter",
                  parents=[src], params={"expr": "is_active == True"})
agg = b.add_idea(name="aggregate", op_id="groupby",
                 parents=[filt], params={"by": "x"})
graph = b.finish()

assert graph.is_acyclic()
graph.validate_audit_monotonic()        # raises ValueError on violation
print(graph.emit_mermaid())             # Quarto-renderable diagram
print(graph.emit_text())                # plain-text tree

# Ancestor traversal
ancs = graph.ancestors(agg)             # sorted list of u64 fingerprints
```

### 2.3 The determinism contract

The thing that distinguishes Locke from Pandera / GE. All true today:

- Finding IDs are content-addressed (SHA-256-derived 64-bit fingerprint over `(code, severity, message, column, evidence)`)
- JSON output is canonical — BTreeMap-sorted keys, no timestamps, no UUIDs, no map-iteration noise
- Numeric reductions use Kahan summation
- `BTreeMap` / `BTreeSet` everywhere; no `HashMap` iteration leak
- RNG is SplitMix64 with explicit seed threading
- SIMD paths avoid FMA (preserves bit-equality across CPU vendors)
- Two `validate(df)` calls produce byte-identical JSON across runs, machines, and OS

The deep-dive audit (Locke v0.7+, all shipped this session) ran four
parallel agent-clusters over ~26K LOC and found exactly four
silent-correctness bugs over the lifetime of the codebase — all
shipped with regression tests in `tests/locke/deep_dive_regression_tests.rs`.

## 7. The one-paragraph elevator pitch

For the PyPI long description, the GitHub repo banner, or any single-paragraph reference, this is the shortest accurate framing:

> **`cjc-locke`** is a deterministic data-quality library for Python: 60+ finding codes (sentinels, drift, leakage, distribution shape, categorical quality, PII) that fire automatically against a `pandas.DataFrame` with zero configuration, producing byte-identical JSON reports across runs and machines so CI gates can `git diff` two runs and trust the result. The Python binding wraps a Rust crate (`cjc-locke`, on crates.io); FFI overhead is one column-copy per call via the numpy buffer protocol, the Rust side runs unchanged so determinism survives intact, and the library has zero hard runtime deps (`numpy`/`pandas`/`polars` are optional and auto-detected). Built on the same audit-chain primitives as the rest of CJC-Lang: content-addressed `FingerprintId`s, monotonic audit events, 8-axis composable belief scores under a proven meet-semilattice algebra. The shipped Rust crate has 408 unit tests + 277 integration tests + 629 audit-chain canaries, all green. MIT-licensed.
