# Cannot assign through an immutable borrow

## What happened

You tried to mutate a value through a borrow that isn't a mutable
borrow. The closely-related `E2010` fires for direct assignment to an
immutable binding; `E3004` fires for assignment via a reference that
isn't `&mut`.

```cjcl
let mut v = [1, 2, 3];
let r = &v;            // immutable borrow
r[0] = 99;             // E3004 -- r is &, not &mut
```

## Why it matters

The "exclusive mutable, shared immutable" rule means immutable borrows
are a promise: "while I exist, the value won't change." Letting an
immutable borrow be used to write would break that promise -- code
holding another `&v` would see the value change underneath them.

CJC-Lang catches this at compile time so the assumption holds. This is
load-bearing for aliasing analysis: if you have `&v` you can rely on
`v` not changing for the borrow's lifetime, without runtime checks.

## How to fix it

**Use a mutable borrow:**

```cjcl
let mut v = [1, 2, 3];
let r = &mut v;        // mutable borrow
r[0] = 99;             // OK
```

**Or assign through the owner directly:**

```cjcl
let mut v = [1, 2, 3];
v[0] = 99;             // OK -- direct mutation, no borrow involved
```

## Common pitfall

The borrow's type (`&` vs `&mut`) determines what you can do with it,
not the underlying binding's mutability. Even with `let mut v`, taking
`&v` gives you a non-mutable borrow that can't be used to write.

If your function takes a borrow and you want it to be able to mutate,
declare the parameter as `&mut T`, not `&T`.
