# An imported module/symbol was never used

## What happened

You wrote `import some_module;` (or `import foo from some_module;`)
but never referenced `some_module` (or `foo`) anywhere in this file.

## Why it matters

Unused imports are usually leftovers from refactoring: you used to call
into `vizor`, then rewrote the section to not need it, but forgot to
delete the import. They're not strictly wrong -- the program runs
correctly -- but they:

- Make the file look like it depends on something it doesn't
- Slow compilation (the imported module has to be loaded)
- Confuse readers about what the file actually uses

CJC-Lang surfaces this as a warning (compilation continues) so you
can clean it up at your convenience without it blocking work.

## How to fix it

Delete the unused import:

```cjcl
// before
import vizor;
import math;

fn main() -> i64 {
    let m = math.mean(values);    // only `math` is used
    print(m);
    0
}

// fix
import math;

fn main() -> i64 {
    let m = math.mean(values);
    print(m);
    0
}
```

If you genuinely want the import for its side effects (e.g., a module
whose load registers something), CJC-Lang doesn't currently support
that pattern -- imports must be referenced. Prefix the binding with
`_` (the unused-binding convention) if your version supports the
`import x as _` form.

## Common pitfall

If you have a multi-file project where `import vizor` makes a name
like `vizor` available, and you reference `vizor.bar_chart(...)`, that
counts as a use. W0002 only fires when there is genuinely no reference
to the imported symbol.
