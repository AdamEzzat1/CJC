# A struct literal is missing a required field

## What happened

You wrote a struct literal but didn't supply a value for every field
declared in the struct. The diagnostic shows which field(s) are missing.

```cjcl
struct Point { x: i64, y: i64 }

let p = Point { x: 1 };    // E2015 -- missing `y`
```

## Why it matters

CJC-Lang structs require **all** fields to be initialized at the
literal. There's no "default" value the compiler can quietly fill in:
the language doesn't have implicit zero/null/None for arbitrary types,
and even if it did, having defaults vary by type would be a hidden
contract that's hard to reason about.

For determinism specifically: an implicitly-defaulted field would have
its value depend on the type's default, which could change as the type
evolves. Explicit initialization keeps the literal's meaning stable
across edits.

## How to fix it

Supply every field:

```cjcl
let p = Point { x: 1, y: 0 };
```

If you have a "default constructor" pattern, build it explicitly:

```cjcl
fn default_point() -> Point {
    Point { x: 0, y: 0 }
}

// then -- shorthand init via spread (if supported)
let p = Point { x: 1, ..default_point() };
```

The `..base` syntax (if implemented in your CJC-Lang version) fills
remaining fields from a base value. Check the language status for
whether this is available.

## Common pitfall

If your struct has many fields and you find yourself writing the same
defaults at every literal site, consider whether the type should have
a helper constructor function. CJC-Lang doesn't auto-generate `new`,
so writing one yourself is the convention.
