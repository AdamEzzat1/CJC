# Expected a function body

## What happened

The parser saw the signature of a function (`fn name(params) -> ret`)
but the next token wasn't `{` (the opening of a body). Common causes:

- A stray semicolon between the signature and the body
- An unintended close-brace earlier that pre-empted the body
- A function declaration meant to be a forward declaration (not
  supported in CJC-Lang)

## Why it matters

CJC-Lang doesn't currently support forward declarations or `extern fn`
-- every function has a definition at the place it's declared. This
keeps the language simple to learn and removes a class of "function
declared but never defined" link errors that affect C-family languages.

If you need to call a function from another file, the answer is the
module system (`import other_module;` -- see also `E9001`), not a
forward declaration.

## How to fix it

**Stray semicolon:**

```cjcl
fn solve(x: i64) -> i64;   // E1006 -- no body
{
    x * 2
}

// fix
fn solve(x: i64) -> i64 {
    x * 2
}
```

**Function meant for another file:**

```cjcl
// in main.cjcl
fn helper(x: i64) -> i64;   // E1006 -- attempting forward decl

// fix -- put the body here, or import from another file
fn helper(x: i64) -> i64 {
    x + 1
}
```

## Common pitfall

If you see `E1006` followed by a cascade of other errors, it often
means an earlier function's body had an extra `}` that closed
prematurely. Look upward for unbalanced braces; the editor's
bracket-matching feature helps a lot.
