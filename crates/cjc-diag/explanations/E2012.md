# Tried to call something that isn't a function

## What happened

You wrote `name(args)` but `name` doesn't refer to a function -- it
refers to a value (an i64, a String, a struct, etc.). The diagnostic
shows the actual type.

```cjcl
let x = 5;
x(10);     // E2012 -- x is i64, not a function
```

## Why it matters

CJC-Lang distinguishes function types from value types at the type
system level. Calling a non-function isn't a "runtime feature" the
way it is in some dynamic languages where everything is callable; it's
a static error.

The reason this matters for determinism: if any value could be called,
the compiler couldn't predict what code would execute at any given
call site -- the call would resolve at runtime, which is exactly the
kind of late-binding that breaks reproducibility.

## How to fix it

**Typo where you meant a different name:**

```cjcl
let solver = make_solver();
let result = solv(5);     // E2012 -- meant `solver(5)` or `solve(5)`?

// fix
let result = solver(5);
```

**Variable shadows a function:**

```cjcl
fn compute(x: i64) -> i64 { x * 2 }

let compute = 5;            // shadows the function with an i64
compute(10);                // E2012 -- compute is now i64

// fix -- rename one
fn compute(x: i64) -> i64 { x * 2 }
let count = 5;
compute(10);
```

**Higher-order function returning a non-function:**

```cjcl
fn pick(b: bool) -> i64 { if b { 1 } else { 2 } }

let f = pick(true);
let v = f(5);     // E2012 -- f is i64

// fix -- pick must return a function for this to work
```

## Common pitfall

When higher-order functions are involved, double-check what the outer
function actually returns. The first-class-functions story in
CJC-Lang requires the type to BE a function type -- you can't call
through an `Any` value without dispatch.
