# Ambiguous name

## What happened

You referenced a name that could resolve to two or more different
declarations in scope. The compiler refuses to pick one -- you must
disambiguate explicitly.

```cjcl
import math;
import statistics;
// both modules export `mean`

let m = mean(values);    // E5002 -- math.mean or statistics.mean?
```

## Why it matters

Implicit "first-declared wins" or "last-imported wins" rules are the
source of subtle bugs across many languages. A small change in import
order can silently switch which implementation is being called, and
the program continues to compile -- you only notice when the output
goes wrong.

CJC-Lang refuses to make this choice for you. Ambiguity is a
compile-time error, and resolving it requires you to write what you
meant.

## How to fix it

**Qualify the name:**

```cjcl
import math;
import statistics;

let m = statistics.mean(values);    // explicit
```

**Or import only the version you want:**

```cjcl
import statistics;
// math.mean not imported -- no ambiguity
let m = mean(values);
```

**Or rename on import (if your CJC-Lang version supports it):**

```cjcl
import math as math;
import statistics as stats;

let m1 = math.mean(values);
let m2 = stats.mean(values);
```

## Common pitfall

This error frequently arises when adding a new dependency that turns
out to define common names. The fix is mechanical (qualify or rename),
but the lesson is to prefer explicit qualification by default in
multi-module programs -- it's harder to trip on later.
