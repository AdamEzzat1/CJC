# An unexpected character appeared in the source

## What happened

The lexer (the first stage of the compiler) ran into a character it
doesn't recognize as part of any CJC-Lang token. The diagnostic points
at the offending character.

This is a **lexer error** -- it fires before the parser ever sees your
source. The most common causes are stray non-printable characters
(often invisible whitespace pasted from another editor), accidental
typing of a symbol the language doesn't use, or a UTF-8 character where
ASCII was expected.

## Why it matters

CJC-Lang's lexer is intentionally narrow: it accepts ASCII letters,
digits, common punctuation, ASCII whitespace, and UTF-8 string contents
inside `"..."`. Anything else is rejected. This is deliberate -- a
lexer that silently accepts unusual characters as identifier-letters
makes it easy for bugs like "homoglyph attacks" (where a Cyrillic `a`
replaces a Latin `a`) to slip in. CJC-Lang refuses to be subtle here.

## How to fix it

**Stray invisible characters:**
Open the file in an editor that shows whitespace and non-printables
(VS Code: View -> Render Whitespace; vim: `:set list`). Delete the
unexpected character.

**Wrong symbol for an operation:**

```cjcl
let x = 5 ÷ 2;     // E0001 -- `÷` is not a CJC operator

// fix
let x = 5 / 2;
```

**Non-ASCII identifier (intentional or not):**

```cjcl
let café = "espresso";   // E0001 on `é` -- non-ASCII letters not allowed

// fix
let cafe = "espresso";   // ASCII-only identifiers
```

## Common pitfall

Pasting code from web pages or PDFs often introduces "smart quotes"
(curly `"..."` or `'...'`), em-dashes, or non-breaking spaces. They all
look correct in a rich-text view but become E0001 in the lexer. The
robust fix is to retype the line, not to copy-paste.
