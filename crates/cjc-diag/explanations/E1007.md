# A struct literal repeats the same field name

## What happened

You wrote a struct literal where one field appears more than once:

```cjcl
let p = Point { x: 1, y: 2, x: 5 };   // E1007 -- `x` appears twice
```

The parser flagged this because there's no defined semantics for
"second value wins" or "last write wins" -- duplicates are simply
disallowed.

## Why it matters

Duplicate field names are almost always typos or copy-paste accidents.
Letting them through with implicit "last write wins" would hide the
bug: the program would compile, then quietly do the wrong thing.
Compile-time rejection forces you to notice.

This also keeps struct-literal semantics straightforward: each field
gets exactly one value, in any order, with no rules to memorize about
which duplicate wins.

## How to fix it

Pick one value and remove the duplicates:

```cjcl
// before
let p = Point { x: 1, y: 2, x: 5 };

// fix -- pick one
let p = Point { x: 5, y: 2 };
```

If you genuinely meant two different fields, check the struct
definition -- you may have intended different field names:

```cjcl
struct Span { start: i64, end: i64 }

let s = Span { start: 0, start: 10 };   // E1007 -- meant `end: 10`?

// fix
let s = Span { start: 0, end: 10 };
```

## Common pitfall

Auto-complete in some editors can insert a duplicate field if you tab
through quickly. Spot-checking the literal before saving catches this.
