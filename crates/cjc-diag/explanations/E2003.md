# Incompatible types in a binary operation

## What happened

You applied a binary operator (`+`, `-`, `*`, `/`, `<`, `>`, `==`, etc.)
to two operands whose types don't form a valid combination for that
operator. The diagnostic shows the operator and the operand types.

```cjcl
let x = 1 + "hello";        // E2003 -- i64 + String not valid
let b = [1, 2] == "ab";     // E2003 -- array vs string comparison
```

This is more specific than `E2001` (general type mismatch): `E2003`
specifically fires when an operator can't accept the operand types
together.

## Why it matters

CJC-Lang doesn't define implicit conversions for binary operators.
Each operator has a fixed set of (left, right) type pairs it accepts:

- Arithmetic (`+`, `-`, `*`, `/`, `%`): `(i64, i64) -> i64`,
  `(f64, f64) -> f64`, plus string concatenation for `+` on `(String, String)`
- Comparison (`<`, `>`, `<=`, `>=`): same-numeric-type pairs
- Equality (`==`, `!=`): same-type pairs
- Logical (`&&`, `||`): `(bool, bool) -> bool`

Anything outside these matrices is rejected. This avoids the swamp of
"what does `1 + true` mean" that some languages embrace.

## How to fix it

Convert one operand to match the other:

```cjcl
// before
let x = 1 + 2.5;          // E2003

// fix -- explicit conversion
let x = (1 as f64) + 2.5;

// or change the literal type
let x = 1.0 + 2.5;
```

```cjcl
// String concatenation needs both sides to be String
let n = 42;
let msg = "count: " + n;   // E2003 -- String + i64

// fix
let msg = "count: " + i64_to_string(n);
// or use formatted strings
let msg = f"count: {n}";
```

## Common pitfall

`bool + bool` and `bool == 1` are common typos when migrating from C or
Python. CJC-Lang doesn't treat `true` as `1` -- the types are distinct.
Use explicit conversion if you need it:

```cjcl
let count = (cond as i64) + base;
```
