# A function name was called but never declared

## What happened

You called a function whose name doesn't refer to any declared
function, builtin, or imported symbol in scope. The diagnostic shows
the name you used.

## Why it matters

Like `E2006` (undefined variable), this is a compile-time check that
protects CJC-Lang's determinism guarantees. A program that resolves
function calls at runtime would have nondeterministic behavior
depending on load order, monkey-patching, or environment.

The distinction between `E2006` and `E2007` is small but real: `E2006`
fires when the name appears as a value reference; `E2007` fires when
the name appears in call position (`name(args)`). Both have the same
root cause -- the name doesn't resolve -- but the diagnostic
classifies them differently so suggestions can be tailored (a missing
function is often fixed differently from a missing variable).

## How to fix it

**Typo:**

```cjcl
print(maen([1.0, 2.0, 3.0]));   // E2007: undefined `maen`

// fix
print(mean([1.0, 2.0, 3.0]));
```

**Missing import (multi-file projects):**

```cjcl
// in main.cjcl
let plot = bar_chart(...);   // E2007 if vizor isn't imported

// fix -- import the module
import vizor;
let plot = vizor.bar_chart(...);
```

**Builtin name you expected to exist doesn't:**

```cjcl
let result = sqrt2(16.0);   // E2007 -- the builtin is `sqrt`, not `sqrt2`

// fix
let result = sqrt(16.0);
```

CJC-Lang has 475+ builtins, but the names are specific. When you're not
sure if something exists, `crates/cjc-runtime/src/builtins.rs` is the
authoritative list.

## Common pitfall

If you're using the multi-file module system (`cjcl run --multi-file
main.cjcl`), every cross-file reference needs an `import` at the top.
CJC-Lang does not implicitly merge namespaces across files.

## Common pitfall (method-like calls)

Some operations look like method calls (`tensor.transpose()`) but are
actually method calls on values, not free functions. If you wrote
`transpose(tensor)` and got E2007, the correct form is
`tensor.transpose()`. The reverse also happens -- some operations are
free functions even though they "feel" methody (`array_push(arr, val)`
not `arr.push(val)`).
