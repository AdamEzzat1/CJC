# Tried to index something that isn't indexable

## What happened

You wrote `value[i]` but the type of `value` doesn't support indexing.
The diagnostic shows the type. Indexable types in CJC-Lang are:

- Arrays: `[T]`
- Tuples: `(A, B, C)` (with `.0`, `.1`, `.2` access, not `[0]`)
- Strings: `String[i]` returns a byte
- Tensors: `tensor[[i, j]]`
- ByteSlices: `bytes[i]`

Anything else -- scalars, structs, functions -- can't be indexed.

## Why it matters

CJC-Lang's indexing is a type-checked operation: the compiler must
know the element type produced by `value[i]` before lowering can
generate the right code. For types where this isn't defined (an `i64`
doesn't have "elements"), the operation is rejected.

This is also what makes CJC-Lang's bounds checks deterministic --
because the index produces a known element type, the compiler can
emit the right OOB check for the right container (see `E8001`).

## How to fix it

**Use the right access syntax for the type:**

```cjcl
let tup = (1, 2, 3);
let first = tup[0];     // E2013 -- tuples use .0/.1/.2

// fix
let first = tup.0;
```

```cjcl
struct Vec3 { x: f64, y: f64, z: f64 }
let v = Vec3 { x: 1.0, y: 2.0, z: 3.0 };
let first = v[0];       // E2013 -- structs aren't indexed

// fix -- use field access
let first = v.x;
```

**Wrong type assumption:**

```cjcl
fn process(value: i64) {
    let bit = value[3];    // E2013 -- i64 isn't indexed; you want bit shifts
}

// fix -- bit-shift to extract a bit
fn process(value: i64) {
    let bit = (value >> 3) & 1;
}
```

**Multi-dimensional tensor needs an array of indices:**

```cjcl
let t: Tensor[f64] = ...;
let v = t[0];          // E2013 if t is multi-dimensional

// fix
let v = t[[0, 1]];     // explicit per-axis indices
```

## Common pitfall

`String[i]` returns a single byte (`u8`), not a character. If you want
the character at position `i`, the language doesn't have direct support
for that (UTF-8 makes it non-constant-time). Iterate using string
methods instead.
