# Double mutable borrow

## What happened

You tried to take a second mutable borrow (`&mut v`) of a value that
already has one in scope. The aliasing rule forbids two simultaneous
mutable borrows -- exclusivity is the whole point.

```cjcl
let mut v = [1, 2, 3];
let a = &mut v;       // first mutable borrow
let b = &mut v;       // E3008 -- can't have two
a[0] = 99;
b[1] = 88;
```

## Why it matters

A mutable borrow promises exclusive write access. Two of them would
let two different code paths mutate the value without coordination,
which is exactly the data-race scenario the aliasing rule exists to
prevent.

For deterministic execution specifically, two concurrent writers
through aliased pointers would make the order of writes
observation-dependent -- which is the antithesis of CJC-Lang's
guarantees.

## How to fix it

**Sequence the borrows so only one is active at a time:**

```cjcl
let mut v = [1, 2, 3];
{
    let a = &mut v;
    a[0] = 99;
}                       // first borrow ends
let b = &mut v;
b[1] = 88;
```

**Use non-lexical lifetimes** (CJC-Lang ends a borrow at its last
use):

```cjcl
let mut v = [1, 2, 3];
let a = &mut v;
a[0] = 99;              // last use of a ends its borrow
let b = &mut v;         // OK now
b[1] = 88;
```

**Or split the borrow** if you need access to different parts:

```cjcl
let mut v = [1, 2, 3];
let (left, right) = v.split_at_mut(1);   // two disjoint mutable borrows
left[0] = 99;
right[0] = 88;
```

The `split_at_mut`-style API gives you two `&mut` borrows that the
compiler knows refer to disjoint memory, so they don't violate
exclusivity.

## Common pitfall

`E3008` and `E3007` cover related rules:
- `E3008` = `&mut` + `&mut` (this code)
- `E3007` = `&mut` + `&` (mutable while immutable exists)
- Two `&` borrows are fine -- shared readers don't conflict.

If you genuinely need multiple mutable references to the same value,
the design pattern is "borrow-cell" / interior mutability (`Cell`,
`RefCell`), which moves the exclusivity check from compile time to
runtime in exchange for the flexibility.
