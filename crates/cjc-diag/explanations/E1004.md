# A closing delimiter is missing

## What happened

The parser encountered an opening brace `{`, paren `(`, or bracket `[`,
but never found the matching closing token before reaching the end of
the file (or before hitting something that couldn't appear inside the
unclosed scope).

## Why it matters

Delimiters in CJC-Lang are paired and balanced. The parser uses them to
know when a block ends, when a function-call argument list ends, when an
array literal ends. Without balance, the parser can't tell where one
construct stops and the next begins, so it stops with this error.

The reason CJC-Lang doesn't *guess* the balance (e.g., by inserting
closes at end-of-line as some scripting languages do) is the same as for
`E1000`: a parser that quietly fixes mismatches can silently change the
meaning of your program. Balance is required.

## How to fix it

Find the opener that didn't get closed and add the matching closer:

```cjcl
fn main() -> i64 {
    let x = compute(
//                  ^ opener never matched
    x
}
// E1004 on EOF

// fix
fn main() -> i64 {
    let x = compute();
    x
}
```

When the diagnostic spans multiple lines, the editor's bracket-matching
feature (most IDEs highlight matched brackets) is usually the fastest
way to find the unmatched opener.

## Common pitfall

The error span often points to **EOF or the closing brace of the
surrounding function**, not the actual unclosed opener. That's because
the parser doesn't know there's an imbalance until it runs out of
source. Look *upward* from where the diagnostic fires for the opener
that has no match.

## Common pitfall (string literals)

A `"` that's never closed can swallow what looks like balanced braces,
because everything between the unclosed quotes is parsed as a string
literal. If E1004 fires on a line that *looks* balanced, scan upward
for a quote that should have been escaped.

A related lexer error (`E0002`, unterminated string) usually fires
first in this case -- if you see both errors in the same compile,
fix `E0002` first; `E1004` may resolve itself.
