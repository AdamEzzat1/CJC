# Unresolved name

## What happened

You referenced a name -- a variable, function, type, or module --
that doesn't resolve to anything in scope. The diagnostic shows the
unresolved name.

This is the umbrella name-resolution failure. The more specific
cousins are:

- `E2006` -- undefined variable (when the name is used as a value)
- `E2007` -- undefined function (when the name is used in call position)
- `E5004` -- import not found (when the name is in an `import`)

E5001 fires when the resolver can't categorize the failure into one
of those, or for less-common reference sites (type names, module
paths, attribute names).

## Why it matters

CJC-Lang resolves every name at compile time. There is no late binding,
no runtime fallback, no implicit `__getattr__`-style lookup. A name
that doesn't resolve at compile time is a name that will never resolve.

This is one of the load-bearing pieces of CJC-Lang's determinism story
-- a program's reference graph is fixed at compile time, not affected
by what other modules happen to be loaded or what the environment
looks like.

## How to fix it

The fix depends on what the name was supposed to be:

**Typo:**

```cjcl
let temperatue = 22.0;
print(temperature);   // E5001 -- you wrote `temperatue` above

// fix
let temperature = 22.0;
print(temperature);
```

**Missing import:**

```cjcl
// referenced but not imported
let plot = vizor.bar_chart(...);   // E5001 if vizor not imported

// fix
import vizor;
```

**Type name not yet declared:**

```cjcl
fn solve(x: Point) -> Point { x }    // E5001 if Point is undefined

// fix -- declare the type or import it
struct Point { x: f64, y: f64 }
```

## Common pitfall

When you see E5001 followed by a cascade of other errors, the
cascading errors usually have the same root cause. Fix E5001 first
and re-run.
