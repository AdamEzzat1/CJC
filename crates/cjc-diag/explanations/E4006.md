# Mutation in a pure context

## What happened

Inside a function declared pure (no `/mut` effect), the compiler
detected a mutating operation -- an assignment to a mutable variable,
a write into an indexed location, a struct field write.

```cjcl
fn solve(x: i64) -> i64 /pure {
    let mut buf = [0, 0, 0];
    buf[0] = x;    // E4006 -- pure functions can't mutate, even local state
    sum(buf)
}
```

## Why it matters

The `/pure` effect is the strongest of CJC-Lang's purity guarantees.
A pure function:

- Has no observable side effects (no IO, no global state writes)
- Doesn't depend on or modify any state outside its arguments
- Returns the same value for the same arguments, every time
- Can be inlined, reordered, deduplicated, or evaluated in parallel
  safely

Allowing internal mutation in a `/pure` function would break the
reorder-safety property: two pure calls that share internal mutable
state could produce different results depending on which ran first.

Note that this is STRICTER than functional-language purity, which
typically allows internal mutation as long as it isn't observable.
CJC-Lang's `/pure` is stricter so the optimizer can reorder
aggressively without proof obligations.

## How to fix it

**Drop the `/pure` annotation if you need internal mutation:**

```cjcl
fn solve(x: i64) -> i64 {
    let mut buf = [0, 0, 0];
    buf[0] = x;
    sum(buf)
}
```

**Or rewrite without mutation:**

```cjcl
fn solve(x: i64) -> i64 /pure {
    let buf = [x, 0, 0];     // build the final value directly
    sum(buf)
}
```

**Use the `/mut` effect for functions that genuinely need local
mutation but no IO:**

```cjcl
fn solve(x: i64) -> i64 /mut {
    let mut buf = [0, 0, 0];
    buf[0] = x;
    sum(buf)
}
```

`/mut` is weaker than `/pure` (allows local mutation) but stronger
than the unannotated default (still no IO).

## Common pitfall

Functional-language migrants expect "no observable mutation" purity.
CJC-Lang's `/pure` is "no internal mutation either." This is more
restrictive but unlocks stronger optimization guarantees. If you find
`/pure` too restrictive, `/mut` is usually the right relaxation.
