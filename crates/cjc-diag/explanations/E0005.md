# A block comment was never closed

## What happened

The lexer found `/*` opening a block comment but reached the end of
the file before finding the matching `*/`. Everything from `/*` to EOF
was being treated as comment content, which usually means a large
portion of your code was silently ignored.

## Why it matters

Block comments in CJC-Lang can span any number of lines and contain
any characters, including `//` (line comment markers don't escape the
block). The only thing that closes a block comment is `*/`. If you
forget it, the lexer can't help you guess where the comment was
supposed to end -- so it stops at EOF and reports the unclosed opener.

The reason CJC-Lang doesn't auto-close at end-of-line (unlike line
comments `//` which always end at newline) is consistency: block
comments are explicitly multi-line by design.

## How to fix it

Find the unclosed `/*` and add the matching `*/`:

```cjcl
/* TODO: write this later
fn main() -> i64 { 0 }   // E0005 at EOF -- whole file is in the comment

// fix
/* TODO: write this later */
fn main() -> i64 { 0 }
```

## Common pitfall

**Nested block comments are not supported.** The first `*/` closes the
outermost `/*` regardless of nesting:

```cjcl
/* outer
   /* inner */
   still part of outer?
*/
```

The lexer sees this as `/* outer ... */` (closed at the first `*/`)
followed by `still part of outer? */` -- two errors on the latter
line. If you need to comment out a region that already contains block
comments, use line comments (`//`) on every line instead, or
temporarily change `/*` inside to something else.
