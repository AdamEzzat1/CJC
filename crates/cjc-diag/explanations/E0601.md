# Snapshot logic hash mismatch

## What happened

You tried to load a `cjc-snap` snapshot, and the hash of the snapshot's
"logic" section (the type/function definitions present at save time)
doesn't match the hash of the corresponding definitions in the
currently-loaded program.

In other words: the program that created the snapshot is structurally
different from the program trying to load it, in ways that affect the
snapshot's interpretation.

## Why it matters

`cjc-snap` is CJC-Lang's binary serialization format. Snapshots embed
hashes of three things:

- **Logic hash** (this code) -- the function/type definitions
- **Type hash** (`E0602`) -- the shape of stored values' types
- **Layout hash** (`E0603`) -- the byte layout used for serialization

These hashes make snapshot loading **safe**: if the program shape has
changed, the loader knows the stored bytes can no longer be
interpreted the way they were saved, and refuses to deserialize
rather than producing garbage values.

For CJC-Lang's audit and reproducibility story, this is critical. A
snapshot from version X reloaded into version Y must produce
identical values, or refuse. Silent reinterpretation is the worst
possible outcome.

## How to fix it

**Recreate the snapshot** with the new program version:

```bash
# old program -- generated snapshot.bin
cjcl run --save-snapshot snapshot.bin old_program.cjcl

# new program -- regenerate the snapshot
cjcl run --save-snapshot snapshot.bin new_program.cjcl
```

**Or revert the program to the version that matches the snapshot.**
Useful if the snapshot is precious (a benchmark run, an audited
model) and you'd rather adjust the program than re-run the work.

**For long-term snapshot storage**, version your snapshots alongside
the program code so the matching version is always available. The
hash mismatch is telling you what would otherwise be a silent failure
-- treat it as a feature, not a bug.

## Common pitfall

Even minor edits can shift the logic hash: adding a function, renaming
a struct, reordering declarations. Don't try to "tweak" your program
to match a snapshot's hash -- the right answer is to rebuild the
snapshot from the new program.
