# Effect annotation mismatch (declared vs actual)

## What happened

A function is declared with one set of effects, but its body performs
operations whose effects aren't included in the declaration. For
example, a function declared `/det` whose body does IO (which is the
`/io` effect):

```cjcl
fn solve(x: f64) -> f64 /det {
    print("x = {x}");    // E4005 -- body performs /io, signature says /det only
    x * 2.0
}
```

This is the **umbrella** effect error -- the body's effect set isn't a
subset of the signature's effect set. The more specific cousins
(`E4001`, `E4002`, `E4003`, `E4004`, `E4006`) fire for the most
common single-effect violations.

## Why it matters

CJC-Lang's effect system enforces that a function's declaration is a
*lower bound* on what it can do -- the body may use any subset of the
declared effects, but no more. This is a one-way relationship that
makes effects predictable to callers.

The reason: callers reason about effects by reading signatures. If a
function declared `/det` could secretly do IO, every `/det` caller
would inherit IO they didn't expect. The effect system would degrade
into a comment.

## How to fix it

**Add the missing effect to the declaration:**

```cjcl
fn solve(x: f64) -> f64 /det,io {
    print("x = {x}");
    x * 2.0
}
```

**Or remove the effect-needing code:**

```cjcl
fn solve(x: f64) -> f64 /det {
    x * 2.0        // no effects beyond /det
}
```

**Or factor IO into a separate function:**

```cjcl
fn solve(x: f64) -> f64 /det {
    x * 2.0
}

fn solve_with_log(x: f64) -> f64 /det,io {
    print("x = {x}");
    solve(x)
}
```

## Common pitfall

Effects are NOT inferred from the body. The signature is authoritative
-- if you declare `/det`, you opt out of every other effect including
ones you might not have realized you were using. The compiler is the
arbiter; trust the error message about which specific operation is
the culprit.
