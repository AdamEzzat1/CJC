# Invalid assignment target

## What happened

You wrote an expression on the left side of `=` (in an assignment, not
a `let` binding) that isn't a place that can be assigned to. Valid
assignment targets are:

- A variable name: `x = 5`
- A field of a value: `point.x = 5`
- An index into a collection: `arr[0] = 5`
- A tuple/struct destructuring (in let bindings, not assignments)

Anything else -- a literal, a function call result, an arithmetic
expression -- can't be assigned to.

## Why it matters

CJC-Lang's assignment is **place-based**: the left side names a
storage location, the right side provides a value, and the assignment
writes the value into the location. Expressions like `x + 1` don't
have a storage location -- they produce a value -- so they can't be
on the left.

This is the same rule as Rust, C, and most other languages, but the
error message in CJC-Lang exists to be specific about what *would* be
a valid target so you can fix it directly.

## How to fix it

**Trying to assign to a literal or expression:**

```cjcl
5 = x;            // E1008 -- 5 is a literal, has no storage

// fix -- you probably meant `x = 5`
x = 5;
```

**Assigning to a function call result:**

```cjcl
get_value() = 10;   // E1008 -- the result is a value, not a place

// fix -- assign to the variable, or mutate via a setter
let mut v = get_value();
v = 10;
```

**Using `==` where you meant `=`:**

```cjcl
if x == 5 = 10 { ... }   // E1008 -- parses oddly

// fix
if x == 5 { x = 10; }
```

## Common pitfall

CJC-Lang's `let` bindings can destructure (`let (a, b) = pair;`) but
plain assignment cannot. If you need to update multiple variables from
a tuple, do them one at a time or use a fresh let:

```cjcl
(x, y) = swap(x, y);          // E1008 -- tuple LHS not allowed in assignment

// fix
let (nx, ny) = swap(x, y);
x = nx;
y = ny;
```
