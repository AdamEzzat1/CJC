# Cannot unify types

## What happened

The type checker was trying to combine two types in a context that
requires them to be the same -- branches of an `if`, arms of a
`match`, elements of an array literal -- and couldn't reconcile them.

```cjcl
let x = if cond { 1 } else { "hello" };
//                ^               ^
//               i64           String     E2004 -- branches must agree
```

## Why it matters

CJC-Lang requires every expression to have **one** type at compile
time. Cases where two expressions reach the same value-position must
agree on type. This is what lets the language pick the right numerical
kernel, allocate the right amount of memory, and produce
bit-identical output across runs.

If `if cond { i64 } else { String }` were allowed, the resulting
"type" would be a runtime union -- the compiler couldn't predict
which type was actually returned at any given call, breaking both
type checking and determinism.

The error is called "unification" because the type checker tries to
**unify** the two types -- find a single type both can be -- and
reports failure when no such type exists.

## How to fix it

**Make all branches agree on type:**

```cjcl
// E2004
let x = if cond { 1 } else { "hello" };

// fix option A -- pick one type
let x = if cond { "1" } else { "hello" };   // both String

// fix option B -- use Any if you genuinely need polymorphism
let x: Any = if cond { 1 as Any } else { "hello" as Any };
```

**For arrays, all elements must agree:**

```cjcl
let arr = [1, 2.0, 3];      // E2004 -- mixing i64 and f64

// fix
let arr = [1.0, 2.0, 3.0];   // all f64
```

**For match arms, all arms must produce the same type:**

```cjcl
match x {
    0 => 0,           // i64
    _ => "other",      // String -- E2004
}

// fix
match x {
    0 => "zero",
    _ => "other",
}
```

## Common pitfall

Empty arrays trigger `E2002` (cannot infer type) rather than `E2004`,
because there are no elements to disagree -- the compiler just can't
guess. Provide a type annotation.
