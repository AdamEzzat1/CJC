# Wrong number of type arguments

## What happened

You wrote a generic instantiation with the wrong number of type
arguments. The diagnostic shows how many the type/function declares
and how many you supplied.

```cjcl
fn solve<A, B>(a: A, b: B) -> A { a }

solve::<i64>(1, "hi");    // E6005 -- expected 2 type args, got 1
```

## Why it matters

Each generic parameter is a slot that the caller fills. If the count
doesn't match, there's an unfilled slot (which can't be inferred from
context) or an extra slot (which the function/type doesn't have).

CJC-Lang doesn't default unfilled slots to `Any` or anything else --
the count must match exactly. This keeps generic instantiation
predictable and unambiguous.

## How to fix it

**Supply the right number of type arguments:**

```cjcl
solve::<i64, String>(1, "hi");    // both filled
```

**Or rely on inference** by omitting the explicit type arguments:

```cjcl
solve(1, "hi");                   // type checker infers <i64, String>
```

The explicit-type-argument form is only needed when inference can't
figure it out (typically because the result type is ambiguous from
context).

## Common pitfall

Methods on generic types often have an "implicit" type argument (the
self type). If you wrote `Vec::<i64>::new()` correctly but it
complains about argument count, double-check whether you're calling
a generic method that has its own type parameters in addition to
the type's.
