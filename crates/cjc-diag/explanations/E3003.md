# Cannot borrow as mutable

## What happened

You tried to take a mutable borrow (`&mut v`) of a value that isn't
mutable in this context. Either the value was bound with `let` (not
`let mut`), or an existing immutable borrow is in the way.

```cjcl
let v = make_vec();      // immutable binding
let r = &mut v;          // E3003 -- v isn't declared mutable
```

## Why it matters

Mutable borrows are exclusive: at any moment, either there is one
mutable borrow OR any number of immutable borrows, never a mix. This
"single writer or multiple readers" rule is what lets CJC-Lang
guarantee data-race freedom and aliasing safety statically.

To take a mutable borrow, the *binding* of the value must itself be
mutable. An immutable binding has promised callers and the rest of
the code that the value won't change -- letting a mutable borrow
break that promise would defeat the model.

## How to fix it

**Declare the binding as mutable:**

```cjcl
let mut v = make_vec();
let r = &mut v;          // OK -- v is now mutable
```

**Or borrow immutably if you only need to read:**

```cjcl
let v = make_vec();
let r = &v;              // immutable borrow -- OK
```

**Or rebind with mutability:**

```cjcl
let v = make_vec();
// later, decide you need to mutate
let mut v = v;           // shadow with a mutable binding
let r = &mut v;
```

## Common pitfall

If you got E3003 inside a function parameter:

```cjcl
fn modify(v: [i64]) {    // v is immutable here
    let r = &mut v;       // E3003
}

// fix -- declare the parameter mutable
fn modify(mut v: [i64]) {
    let r = &mut v;
}
```

Note that mutability of parameters is local to the function -- the
caller's binding isn't affected by what the function does to its own
copy.
