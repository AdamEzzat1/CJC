# Mutable borrow while immutable borrow exists

## What happened

You tried to take a mutable borrow (`&mut v`) of a value that already
has at least one immutable borrow (`&v`) in scope. The borrow
checker rejects this -- the "single writer or multiple readers" rule
forbids any mutable borrow while *any* immutable borrow exists.

```cjcl
let mut v = [1, 2, 3];
let r = &v;           // immutable borrow
let w = &mut v;       // E3007 -- can't take &mut while &v exists
print(r);
print(w);
```

## Why it matters

The aliasing rule is binary: a borrow is either exclusive (`&mut`,
the writer) or shared (`&`, a reader), never mixed. This rule is what
lets the compiler reason about which references can observe which
mutations, and makes data-race freedom a compile-time guarantee.

If a mutable borrow could coexist with immutable ones, the immutable
borrowers could see the value change mid-read, breaking the "value
is stable for the borrow's lifetime" contract.

## How to fix it

**Drop the immutable borrow first:**

```cjcl
let mut v = [1, 2, 3];
{
    let r = &v;
    print(r);
}                       // r dropped
let w = &mut v;         // OK -- no immutable borrow active
```

**Or restructure so the lifetimes don't overlap:**

```cjcl
let mut v = [1, 2, 3];
let r = &v;
print(r);               // last use of r ends its borrow (non-lexical lifetimes)
let w = &mut v;         // OK
```

CJC-Lang uses **non-lexical lifetimes** (NLL): a borrow ends at its
last use, not at the end of its lexical scope. The above pattern works
because by the time `let w = &mut v` happens, `r` has been used for
the last time.

**Or clone:**

```cjcl
let mut v = [1, 2, 3];
let r_owned = v.clone();
let w = &mut v;         // OK -- r_owned is independent
print(r_owned);
print(w);
```

## Common pitfall

If a function takes `&self` and you want to call a method that takes
`&mut self`, you'll hit E3007. The fix is to restructure the call so
the immutable borrow isn't held across the mutable one, or to convert
the design (often via interior-mutability primitives like
`RefCell`, where applicable).
