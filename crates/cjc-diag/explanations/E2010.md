# Cannot assign to an immutable variable

## What happened

You tried to write to a binding declared with `let` (the default --
immutable) rather than `let mut` (explicitly mutable). The diagnostic
points at the assignment and the original `let`.

```cjcl
let x = 5;
x = 10;     // E2010 -- `x` was declared with `let`, not `let mut`
```

## Why it matters

CJC-Lang follows the Rust convention: bindings are **immutable by
default**. You must opt in to mutation with `let mut`. The reason:
most bindings in well-written code don't change after their initial
value, and making mutation explicit makes the mutable ones easier to
audit.

For a deterministic language, this is also a debugging asset. When a
value is wrong, the immutable bindings can be ruled out as the source
of corruption -- they had exactly one assignment. The mutable ones
are the only candidates.

## How to fix it

**Add `mut` to make the binding mutable:**

```cjcl
// before
let x = 5;
x = 10;     // E2010

// fix
let mut x = 5;
x = 10;     // now allowed
```

**Or compute the final value once instead of reassigning:**

```cjcl
// stylistically preferred when possible
let x = if cond { 5 } else { 10 };
```

**Or rebind with a new `let`:**

```cjcl
let x = 5;
let x = x + 1;     // not assignment -- a NEW binding that shadows the old
```

The last form (shadowing) is idiomatic for "transform a value through a
sequence of steps" -- it gives you `let mut` ergonomics without
declaring anything mutable.

## Common pitfall

This is `E2010`, distinct from `E1008` (invalid assignment target).
`E1008` fires when the *form* of the left-hand side is wrong (literal
on LHS, function call on LHS). `E2010` fires when the form is fine
but the binding's mutability disallows the write.
