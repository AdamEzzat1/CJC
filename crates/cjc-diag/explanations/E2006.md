# A name was used that hasn't been bound in this scope

## What happened

You referenced a name (a variable, parameter, or loop binding) that
doesn't exist in any enclosing scope at this position. The diagnostic
shows the unresolved name.

## Why it matters

CJC-Lang resolves names at compile time, not runtime. Every reference
must point to something declared earlier in the same scope or in a
containing scope. There's no implicit "look it up at runtime" fallback
-- if the name doesn't resolve at compile time, the program won't
compile.

This is one of the load-bearing pieces of CJC-Lang's determinism
guarantee. A name that resolves at runtime would behave differently
depending on which other modules happened to be loaded, what order they
ran in, what definitions they shadowed -- all of which would break the
"same seed, same output" promise.

## How to fix it

The fix depends on which case applies:

**Typo:**

```cjcl
let temperature = 22.0;
print(temprature);   // E2006: undefined `temprature` (typo)

// fix
print(temperature);
```

**Out of scope:**

```cjcl
fn outer() {
    let x = compute();
}

fn other() {
    print(x);   // E2006: `x` lives only inside `outer`
}

// fix -- pass it explicitly or compute it here
fn other(x: i64) { print(x); }
```

**Used before declared:**

```cjcl
print(x);            // E2006: `x` is declared below
let x = 1;

// fix -- declare first
let x = 1;
print(x);
```

## Common pitfall

CJC-Lang **does not hoist** declarations the way JavaScript hoists
`var`. Every `let` is in scope only from its declaration site to the
end of the enclosing block. If you reorder code, the order of
declarations matters.

## Common pitfall (closures)

A closure captures variables by name from its defining scope. If you
write a closure that references a name not in scope where the closure
is *defined* (even if the name exists where the closure is *called*),
you'll get E2006:

```cjcl
fn make_printer() {
    // E2006 -- `name` is not in scope here
    print(name);
}
```

The fix is to pass needed values in explicitly or accept them as
parameters.
