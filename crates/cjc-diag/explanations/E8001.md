# Tried to read past the end of a collection

## What happened

Your program ran an indexing operation — `arr[i]`, `tensor[[i, j]]`,
`string[i]` — where the index was outside the collection's valid range.
The diagnostic shows both the index that was requested and the
collection's actual length, so you can see how far off it was.

This is a **runtime** error — meaning your program got far enough to
*try* the operation before crashing. Catching it at compile time isn't
always possible, but it's almost always preferable, and there are
patterns below for doing exactly that.

## Why it matters

Unlike C (undefined behaviour, possible memory corruption) or Python
(`IndexError`, but only after potentially expensive setup), CJC-Lang
treats out-of-bounds access as a clean, deterministic runtime error:
your program halts immediately with a precise message. That's the
correctness story.

But correctness-at-runtime is a *fallback*. The interesting question is
why your code didn't already know the index was safe. Most OOB errors
have a small set of root causes:

- **Off-by-one** — using `arr.len()` where `arr.len() - 1` was meant, or
  vice versa.
- **Stale length cached** — `let n = arr.len()` taken before a loop that
  modifies `arr`.
- **Implicit assumption** that two collections have the same length, which
  silently breaks when one diverges.

Each of these has a structural fix that makes the error impossible, not
just caught.

## How to fix it

**Prefer iteration over indexing.** If you're walking a collection, use
`for x in arr` — there's no index to get wrong:

```cjcl
// before — manual index, easy to break
let i: i64 = 0;
while i < arr.len() {
    let val = arr[i];   // potential E8001 if loop bound drifts
    ...
    i = i + 1;
}

// after — no index variable at all
for val in arr {
    ...
}
```

**Use `.get(i)` when you genuinely need an index but can't be sure it's
in range.** It returns an `Option`, forcing you to handle the missing
case explicitly:

```cjcl
let val = match arr.get(i) {
    Some(v) => v,
    None => default_value,
};
```

**If you really do want to index directly, guard with a bound check:**

```cjcl
if i < arr.len() {
    let val = arr[i];
    ...
} else {
    // handle the missing case
}
```

## Common pitfall

**0-indexed bounds.** Valid indices for `arr` are `0`, `1`, ...,
`arr.len() - 1`. The upper bound is *exclusive*, so a loop that runs
`while i <= arr.len()` is one step too long. The condition for safe
indexing is `i < arr.len()`, not `<=`.

**Multi-dimensional tensors.** `tensor[[i, j]]` requires *both* `i` and
`j` to be in range for their respective axes. The diagnostic tells you
which axis was out — read it carefully when the indices look "almost
right" at a glance.

**Length-coupled arrays.** If you compute `let n = arr.len()` and then
push to `arr` inside a loop, `n` no longer matches the array's actual
length. Either recompute the length each iteration, or iterate the array
directly so the language tracks the length for you.
