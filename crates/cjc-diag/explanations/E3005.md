# Value does not live long enough

## What happened

You created a reference (`&v`) to a value, but the value goes out of
scope before the reference is last used. The borrow checker rejects
this because the resulting reference would be dangling -- pointing at
memory whose owner has already been dropped.

```cjcl
let r;
{
    let v = make_vec();
    r = &v;              // r borrows v
}                        // v dropped here, but r still in scope
print(r);                // E3005 -- r outlives v
```

## Why it matters

References in CJC-Lang are not garbage-collected. Every reference must
have a lifetime that's entirely contained within the lifetime of the
value it points to. The borrow checker tracks lifetimes statically and
rejects any case where a reference could outlive its target.

This is what makes the language safe to use without runtime
reference-counting or GC: references that survive their referent are
impossible by construction, so dangling-pointer bugs don't happen.

## How to fix it

**Restructure so the value lives at least as long as the reference:**

```cjcl
let v = make_vec();         // v's scope extends to end of function
let r = &v;
print(r);                   // OK -- v still alive
```

**Move the value if you need it elsewhere:**

```cjcl
fn outer() {
    let v = make_vec();
    inner(v);               // move v into inner
}

fn inner(v: [i64]) {
    let r = &v;             // r borrows v, which is now owned by inner
    print(r);
}
```

**Or clone to get an owned value:**

```cjcl
let r;
{
    let v = make_vec();
    let r_clone = v.clone();   // r_clone owns a copy
    // do something with r_clone
}
```

## Common pitfall

Returning a reference to a local variable always triggers `E3005`:

```cjcl
fn make_ref() -> &[i64] {
    let v = [1, 2, 3];
    &v                       // E3005 -- v dies when this function returns
}
```

Functions that need to return references must either return references
to inputs they received (lifetime tied to the input) or return owned
values, not references to internal locals.
