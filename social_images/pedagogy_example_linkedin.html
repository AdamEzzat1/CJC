E1003 [parser] · missing type annotation on function parameter
────────────────────────────────────────────────────────────────────────────────────

Function parameters need explicit types

## What happened

You wrote a function whose parameter has no type annotation, like this:

fn solve(x) {
    x * 2
}

CJC-Lang stopped at x because it couldn't tell what kind of value x is supposed to be — an integer, a float, a tensor, something else.

## Why it matters

CJC-Lang treats parameter types as part of the determinism contract. Two reasons this is enforced, not just suggested:

  1. Different types reduce differently. A sum of f32 values and a sum of f64 values give different bit patterns even for the same logical numbers — silently breaking the "same seed, same output" guarantee.
  2. Different types pick different kernels. The compiler routes f64 through Kahan-compensated summation, i64 through overflow-checked paths. Without the annotation it can't pick.
